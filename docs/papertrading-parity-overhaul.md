# Paper-Trading ↔ Backtest Parity Overhaul

**Branch:** `feat/papertrading-realworld-parity`
**Started:** 2026-06-26
**Objective:** Make paper trading a *perfect real-world replica* of backtest results so strategy decisions are trustworthy. Same signals, entries, exits, sizing (kelly/atr/fixed/…), stops/TP/trailing/time-stops, fees, slippage, and PnL — plus a chart with indicators, full-history trigger points (even before the strategy went live), all trades as markers, and active stop/TP lines.

## Root finding (verified by two deep code-survey workflows)

Paper (`forven/scanner.py`) is a **parallel reimplementation** of the backtest (`forven/strategies/backtest.py`); they agree almost nowhere. Verified divergences:

- **Promotion is unreal.** Every gauntlet gating step backtests strategies **full-notional, no stops, ungated**, on a hand-maintained `_vectorized_signals` copy. `post_backtest_submit` reads sizing/stops only from the request body, never from `params['execution_profile']`, and **no backend code ever persists an execution_profile**. So promoted strategies were scored on something you'd never trade.
- **Signal drift:** 10 of 16 builtin types differ between the backtest's vectorized copy and the class `generate_signal` paper runs. Severe: `ichimoku` (backtest = 0 trades), `supertrend` (different algorithm), `williams_r` (inverted ADX). Causes: always-on volume filter in the backtest copy + ADX min/max canonicalization mismatches.
- **Entry:** backtest fills next-bar open; paper fills signal-bar close.
- **Exits:** backtest is intrabar vs high/low, fills at the level, supports trailing + time-stop; paper triggers on a closing-price percent breach, fills at close, no trailing/time-stop, and reads stops from a different param location than sizing (so profile-only stops are invisible to it).
- **Costs:** backtest nets fees+slippage; paper stores gross and sizes off a gross, fee-blind equity.
- **Data:** backtest reads **Binance spot**; paper + chart read **HyperLiquid perps**.
- **Chart:** all four requirements fail — indicators are a third reimplementation (simple-MA, guessed periods); no full-history triggers for arbitrary strategies; trade markers limited to last 500 bars; active SL/TP lines never drawn.

## Locked decisions (user, 2026-06-26)

1. **Scope = unify all + re-score now.** One realistic engine for backtest, gauntlet, and paper; re-baseline every promoted strategy (some demoted).
2. **Price source = HyperLiquid perps** (the live venue). Backtest moves onto the traded series; ingest HL history; remove the silent Binance fallback.
3. **Entry fill = next bar's open** (signal on closed bar i → fill at open of i+1), both engines.
4. **Costs = net** of fees + slippage; sizing-equity is net too; use the venue fee schedule.

## Target architecture — one shared execution kernel

New module `forven/strategies/execution_kernel.py`: a deterministic engine that, given an ordered OHLCV frame + a strategy + a normalized execution profile, produces trades, an equity curve, and markers (filled + unfilled triggers). It owns: signal (single source = `strategy.generate_signals`), next-bar-open entry, sizing (`sizing.py`), intrabar stop/TP/trailing/time-stop exits with the fill conventions, net fee+slippage drag, and one PnL/equity convention (net, leverage-consistent, sizing off running net equity → naturally compounds and matches a compounded backtest).

- **Backtest** runs the kernel over the full history (replaces `_run_directional_signal_series`, `_run_directional_signal_series_with_controls`, `_run_vectorized_backtest`, and the slow per-bar path with this one kernel).
- **Scanner (paper/live)** advances the *same* kernel one closed bar at a time, persisting position/trailing/bars_held/equity state between calls.
- **Acceptance = a replay parity test:** for a representative strategy set, the backtest over a fixed window and a bar-by-bar scanner replay over the same bars must produce trade-for-trade identical results (entry/exit time, price, size, pnl within epsilon). This test *is* the definition of done.

## Round 2 — real-world parity fixes (operator-reported breakage)

A forensic re-audit (8-agent workflow, verified against live code, NOT the "all phases complete" claims) found the engine machinery was sound but fed broken inputs. Fixes landed:

1. **The "$100 on a $10k portfolio" bug — FIXED (shared layer).** `sizing.default_controls()` was `fraction` mode with **no stop**, so `size_fraction` collapsed to `clamp01(risk_per_trade)` = 1% *notional*. Changed the default risk engine to **`atr` mode** (1% risk over a 2x-ATR stop, with a 3% fixed-stop floor when ATR is unavailable) — it now both risk-sizes AND places a real `stop_price`. Lives in shared `sizing.py`, so backtest + paper change together (parity preserved). The legacy scanner path now derives the same 2x-ATR stop via the shared default (was an ad-hoc 1.5x-ATR floor → 33x divergence resolved). `forven/strategies/sizing.py`, `tests/test_sizing.py`.
2. **Risk-engine SELECTION + persistence — ADDED (user decision: risk-adjusted return).** New shared `forven/strategies/execution_selection.py::select_execution_profile` sweeps the four engines {fraction, atr, kelly, full} through the kernel and picks the best by **Sharpe** (sortino/calmar honored when present), with drawdown / min-trade guards. The gauntlet's **paper-promotion gate** now calls it and FREEZES the winner into `params['execution_profile']` before the strategy is param-locked (`gauntlet/tasks.py::_select_and_persist_execution_profile`, gated by `paper_select_execution_profile`, idempotent, best-effort). Paper already reads that profile via `execution_controls_from_params` → it adheres to the chosen engine. `scripts/assign_execution_profiles.py` rewritten as a thin wrapper over the SAME selector (Sharpe objective) to backfill the existing population. `tests/test_execution_selection.py`.
3. **Chart — industry-standard markers.** Triggers now render as muted green up / red down **arrows** (were grey circles); trades render as four distinct LABELED markers **BUY / SELL / SHORT / COVER** (`markerText` no longer returns undefined); an open position draws a solid blue **ENTRY** price line; SL/TP render **dashed** (red/green) with axis labels. Backend bundle is now self-describing (per-marker side/action/shape/color; `active_levels.entry`; typed levels) + indicator-overlay builders added for bollinger_reversion/orb/composites. `api_domains/paper.py`, `chart/ChartWorkspace.svelte`, `trading/PaperTrades.svelte`, `api/paper.ts`, `stores/chartStore.ts`, `tests/test_paper_chart_bundle.py`.
4. **No-auto-fallback hardening (parity stance).** The scanner silently dropped to the divergent legacy per-bar engine on transient errors AND for non-vectorizable strategies. Now: a transient kernel failure returns `KERNEL_SKIP_SCAN` → the scan is skipped (never legacy); a genuinely non-vectorizable paper strategy is flagged `non_vectorizable_no_parity` and NOT traded on legacy unless `paper_legacy_fallback_enabled` (default off). `forven/scanner.py`, `tests/test_scanner.py`.
5. **Funding parity.** Paper now charges perp funding into closed-trade PnL via the backtest's own `_apply_funding_to_trades`, gated by the SAME `backtest_include_funding` setting — held-perp PnL matches the backtest. `forven/scanner.py`.
6. **Downtime tolerance.** The hard 2-bar recording window (dropped trades on any multi-bar scanner outage) is now a configurable `paper_kernel_backfill_bars` (default 6); still bounded so a fresh book never replays full history. `forven/scanner.py`.
7. **Venue decision = "stay Binance + reconcile."** Backtest stays on Binance; `source_reconciliation` now defaults **enabled** (fail-open until a divergence is measured) so the Binance↔HyperLiquid gap is flagged + blocks promotion above `max_divergence_pct`. `forven/dataeng/settings.py`, `forven/policy.py`.
8. **Regime gate** verified already consistent: paper, the confirmation backtest, and the new selector all run `regime_gate=False` (only the WFA robustness leg uses True) — no change needed.

## Round 3 — pre-release correctness audit (17-agent workflow + adversarial verify)

A pre-ship audit (17 specialist auditors across the papertrading + gauntlet math/logic surface, each finding adversarially verified by a math-derivation skeptic + a real-world-impact skeptic, plus a completeness critic) found **49 confirmed findings + 3 critic gaps**. The engine machinery was sound; the defects were subtle guard/metric/scale errors. All actioned:

- **execution_selection (CRITICAL):** the max-drawdown guard read `max_drawdown` but the kernel emits `max_drawdown_pct`, so it was a no-op (never disqualified a catastrophic-DD/full-notional profile). Now reads the right key, fails closed, and falls back to the conservative default when nothing is eligible; degenerate zero-size kelly can't win; `--objective sortino/calmar` no longer silently scores by Sharpe (Sortino surfaced top-level; calmar uses a return/|dd| proxy).
- **execution_kernel + byte-identical backtest copy:** a signal-driven exit (fills at the open) now PRE-EMPTS a same-bar intrabar stop/TP (it was evaluated after); ATR for entry sizing/stops now reads the last CLOSED bar (`signal_idx`) instead of the entry bar's own range (a lookahead). Both copies changed in lockstep; `size_fraction_raw` added so funding scales at full precision.
- **backtest metrics:** Sortino downside deviation corrected to target semideviation about 0 (was mean-centered `np.std`, overstating Sortino); `_compute_backtest_months` made timeframe-aware (was hardcoded hourly → wrong CAGR on other timeframes); WFA aggregate OOS metrics re-based on the summed OOS bar span (was the full IS+OOS window → understated pooled Sharpe); zero-bar trade no longer marks funding incomplete (was blocking promotion).
- **default_leverage:** ONE operator-configurable setting (default 1×) now resolved by the gauntlet confirmation/robustness backtests, execution-profile selection, walk_forward, and the live/paper scanner — ending the 3×-vs-1× split that broke leverage-sensitive (full/fixed/kelly) parity.
- **scanner paper/live parity:** orphan-close `window_start` anchored to the kernel's first replayable bar (warmup+1) so a long-held open isn't wrongly converge-closed; trade_mode resolved via the shared `resolve_backtest_trade_mode`; kernel drag sourced from `backtest_fee_bps` (not `risk_fee_bps`); sizing `initial_capital` pinned to the sandbox base; legacy-live fraction sizing no longer invents a stop distance (was ~33× oversize).
- **indicators (TradingView/Binance parity — REQUIRES RE-BASELINE):** RSI now Wilder-RMA (was SMA/Cutler's); Bollinger/bbwidth/stddev use population std ddof=0; VWAP anchored resets daily; MFI clamps to 100 on all-up windows. Mirrored in the scanner copies so per-bar↔vectorized parity holds.
- **gauntlet:** quick-screen `total_return` normalized to percent-points before comparison; execution-profile reselection made idempotent on the recorded marker (was re-running the full sweep every gate retry when the default engine won); robustness↔selected-profile ordering documented.
- **robustness suite:** integer param jitter forced to a ±1 step when it rounds back to itself (small-int no-op); jitter degradation now scored against an unperturbed SAME-window reference (not the full-window persisted baseline); regime-split requires `regime_split_min_trades_per_regime` (default 5) per regime to count; MC `sharpe_distribution` no longer bogusly annualizes per-trade returns by √252.
- **policy gates:** relaxed preset now lists `min_is_sharpe` (was leaking strict's 0.2 after a switch); `_to_percent_points` no longer under-scales >100% returns; `monte_carlo_percentile_min` + `regime_split_profitable_min` added to the ratio-coercion paths; quick-screen robustness floor reads the editable knob (matches the documented default-0 intent).
- **DSR + optimizer:** `compute_strategy_dsr` uses the scale-invariant `return_pct` (not compounded dollars); optimizer persists the ACTUAL evaluated-combo count as `n_trials` (deflation breadth); `objective=profit_factor` maps +∞ to a large finite sentinel (was ranked last); evolution's apply path merges `{**current, **best}` (was overwriting params with the swept overrides only); **the optimizer's WFA now sizes its fold count so folds actually evaluate** (default 5 folds over 1440 bars gave 201-bar IS slices < the 230 floor → every fold skipped → every optimization candidate rejected at precheck) and the worker returns an explicit "no evaluable folds" error (distinguishable from a robustness FAIL).
- **critic gaps:** manual partial-close on a kernel-managed position is refused (it would double-count when the kernel re-closes the full size); chart trigger replay now uses the strategy's frozen execution profile so the triangles match real paper exits.
  - **KNOWN ISSUE (deferred, not changed):** the paper→live promotion gate compounds `COALESCE(net_pnl_pct, pnl_pct)` as an equity-fraction, but the kernel close writes `net_pnl_pct` as equity-fraction while the legacy/manual `_close_trade_db` path writes a MARGIN return (the H6 funding convention) — so for the RARE non-kernel paper closes the gate's compounding can over-state ~1.5–3×. A first attempt to unify on equity-fraction was REVERTED because it conflicts with the deliberate margin-scale `net_pnl_pct`/`funding_pct`/`fees_pct` convention (the `TestFundingNetPnlH6` contract) and needs a dedicated design pass (unify the kernel + legacy conventions and update all readers together). Practical exposure is small: the orphan-window fix (above) removed the dominant non-kernel paper close in normal kernel-managed operation, and the kernel close — the dominant path — already writes the correct equity-fraction.
- **test integrity:** fixed two stale "mirror" tests that pinned removed behavior (WFA degradation for IS≤0; the MC percentile_rank gate that the verdict no longer uses); added independent regressions for `round_trip_drag` with non-zero costs, the kernel exit precedence + ATR no-lookahead, a non-zero Sortino, the DD guard reading the real metric key, and the `default_leverage` resolver.
